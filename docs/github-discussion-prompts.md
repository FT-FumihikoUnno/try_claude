## Claudeとの対話

### 🤖 @claudeメンションの使い方

#### Issue/PRでの質問
\`\`\`markdown
@claude この実装についてアドバイスをください。

現在の状況:
- ...

質問:
1. ...
2. ...
\`\`\`

#### コードレビューの依頼
\`\`\`markdown
@claude このPRをレビューしてください。

特に以下の点に注目してください:
- コードの品質
- ドキュメントの分かりやすさ
- セキュリティ上の懸念
\`\`\`

#### 実装の提案
\`\`\`markdown
@claude 以下の機能を実装してください。

要件:
- ...
- ...

制約:
- ...
\`\`\`

### 💡 効果的な質問のコツ

1. **具体的に質問する**
   - ❌ 「これ、どう思いますか?」
   - ✅ 「このディレクトリ構成で、拡張性は十分でしょうか?」

2. **コンテキストを提供する**
```markdown
   @claude 
   
   ## 背景
   現在、〇〇という課題があります。
   
   ## 質問
   △△するには、どのアプローチが最適でしょうか?
```

3. **期待する形式を明示する**
```markdown
   @claude プロンプト集の構成案を、箇条書きで3つ提案してください。
```

### 🎯 ユースケース別の例

#### ドキュメント作成

2. セットアップ手順
3. 使い方
4. ディレクトリ構成
\`\`\`

#### バグ調査
\`\`\`markdown
@claude このエラーの原因を調査してください:

\`\`\`
[エラーメッセージ]
\`\`\`

発生条件:
- ...
\`\`\`

#### リファクタリング提案
\`\`\`markdown
@claude この関数をより読みやすくリファクタリングしてください。

現在のコード:
\`\`\`javascript
[コード]
\`\`\`
